Docker-compose del progetto

Il docker-compose.yaml Ã¨ un file che coordina i microservizi al suo interno.

Il servizio init-volume si occupa di creare le sottocartelle necessarie per il funzionamento del progetto e assegnare i permessi.

init-volume: # servizio temporaneo per la creazione delle sottocartelle
    image: alpine
    command: | #shell      
      mkdir -p /shared_data/csv /shared_data/videos /shared_data/frames &&
      addgroup -g 1000 appgroup &&
      adduser -D -u 1000 -G appgroup appuser &&
      adduser -D -u 1001 -G appgroup nodeuser &&
      chown -R 1000:1000 /shared_data &&
      chmod -R 775 /shared_data &&
      chmod g+s /shared_data     
    volumes: 
      - shared_data:/shared_data
    restart: "no"

Il servizio backendkp si occupa di gestire il backend del progetto. Espone la porta 3000 sia all'esterno che all'interno della rete docker. Effettua l'healthcheck con un endpoint /health per verificare il buon funzionamento del servizio. Carica le variabili d'ambiente dal file .env all'interno del servizio Assegna i volumi. Attende che il servizio init-volume si avvii Assegna la rete kpnet Il comando restart riavvia il servizio in caso di errore a meno che non sia stoppato manualmente

backendkp:
    build: ./Frontend
    ports: # porte esposte IN:OUT
      - "3000:3000"
    healthcheck:
      test: ["CMD-SHELL", "curl -f http://localhost:3000/health || exit 1"]
      interval: 10s
      timeout: 5s
      retries: 3
      start_period: 30s
    env_file:
      - .env
    volumes:
      - shared_data:/app/shared_data
    depends_on: # servizi da cui dipende e dopo cui si avvia
      - init-volume
    networks: # rete condivisa
      - kpnet
    restart: unless-stopped # riavvia a meno che non stoppato manualmente

Il servizio extractkp si occupa di estrarre i keypoints dai video. NON espone alcuna porta. E' accessibile SOLO dai servizi interni della rete kpnet. Effettua l'healthcheck con un endpoint /health per verificare il buon funzionamento del servizio. Assegna i volumi condivisi. Attende che il servizio init-volume si avvii Assegna la rete kpnet Il comando restart riavvia il servizio in caso di errore a meno che non sia stoppato manualmente

extractkp:

    #ports: # porte (non) esposte
    #  - "8000:8000"

Il servizio caddy si occupa di gestire il proxy reverse.

Espone le porte 80 e 443 sia all'esterno che all'interno della rete docker. La porta 443/udp si occupa di gestire le richieste di ultima generazione HTTP/3

Assegna i volumi specifici di caddy Effettua l'healthcheck con un endpoint /healthz per verificare il buon funzionamento del servizio. Attende che il servizio backendkp si avvii Assegna la rete kpnet Il comando restart riavvia il servizio in caso di errore a meno che non sia stoppato manualmente

caddy:
    image: caddy:latest
    cap_add:
      - NET_ADMIN # privilegi admin necessari per le porte UDP
    ports:
      - "80:80"
      - "443:443"
      - "443:443/udp" # HTTP/3
    volumes:
      - ./config/caddy:/etc/caddy:ro # ro=read-only bind mount
      - caddy_data:/data # certificati e chiavi
      - caddy_config:/config # configurazione e stato
    healthcheck:
      test: ["CMD-SHELL", "wget -qO- http://localhost/healthz || exit 1"]
      interval: 10s
      timeout: 5s
      retries: 3
      start_period: 30s
    depends_on:
      - backendkp
    networks:
      - kpnet    
    restart: unless-stopped
