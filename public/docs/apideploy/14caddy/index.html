Caddy

Caddy è un servizio di reverse proxy che si occupa di rendere accessibile il web-service all’esterno tramite domini specifici e permette di gestire la sicurezza delle richieste.

In particolare si occupa di:

Healthcheck

Gestisce il proprio healthcheck, utilizzando il matcher :80 per gestire le richieste HTTP e HTTPS.

Le richieste sulla porta 80 arrivano su HTTP e redirezionate su HTTPS. La dicitura permanent dice al browser inviare tutte le successive richieste per questo dominio su HTTPS invece che HTTP, in modo da ottimizzare.

# HealthCheck
:80 {
    handle /healthz {
        respond "OK" 200
    }
    handle {
        redir https://{host}{uri} permanent
    }
}

Reverse proxy

Il cuore di Caddy:

  • Gestisce le richieste preflight OPTIONS per il CORS
  • Controlla il formato delle richieste Solo richieste con quegli headers e metodi sono accettate
  • Redireziona il traffico verso il servizio backendkp su porta 3000
# Produzione (HTTPS automatico)
ai.ortesiriabilitazione.it, www.ai.ortesiriabilitazione.it {
    # Matcher per origini prefetch
    @preflight_ai {
        header Origin "https://ai.ortesiriabilitazione.it"
        method OPTIONS
    }
    @preflight_www {
        header Origin "https://www.ai.ortesiriabilitazione.it"
        method OPTIONS
    }

    # Matcher per tutte le altre origini con metodi != OPTIONS
    @origin_ai {
        header Origin "https://ai.ortesiriabilitazione.it"
    }
    @origin_www {
        header Origin "https://www.ai.ortesiriabilitazione.it"
    }

    # Gestione preflight OPTIONS
    handle @preflight_ai {
        respond "" 204
    }
    handle @preflight_www {
        respond "" 204
    }

    # Settings degli headers in base al matcher
    header @origin_ai Access-Control-Allow-Origin "https://ai.ortesiriabilitazione.it"
    header @origin_ai Access-Control-Allow-Methods "GET, POST, PUT, DELETE, OPTIONS"
    header @origin_ai Access-Control-Allow-Headers "Content-Type, Authorization"

    header @origin_www Access-Control-Allow-Origin "https://www.ai.ortesiriabilitazione.it"
    header @origin_www Access-Control-Allow-Methods "GET, POST, PUT, DELETE, OPTIONS"
    header @origin_www Access-Control-Allow-Headers "Content-Type, Authorization"

    # Reverse proxy
    reverse_proxy backendkp:3000
}
